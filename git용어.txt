<git 용어>
1. git init
- 해당 폴더를 git으로 관리하겠다 라고 지정

2. git add .
- 로컬의 변경사항을 스테이지에 올린다.

3. git commit -m "project init"
- 변경사항을 적용

4. git branch -M main
- 브랜치 이름을 main으로 설정

5. git remote add origin 주소
- github의 저장소와 연결

6. git push origin main
- 로컬의 변경사항을 github에 push(밀어넣음)
